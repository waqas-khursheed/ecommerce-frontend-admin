# Ecommerce Admin — Frontend

Next.js admin panel for the ecommerce backend (`../backed`). The API integration plumbing (HTTP
client, services, auth, route protection) is wired up and working. The UI layer (all 19 pages
listed below) is built with **static/mock data only** — no page except `/login` calls the real
backend yet. Swapping mock data for the real `services/*` calls is the next step.

## Stack / what was installed

- **Next.js 16.2.10** (App Router, Turbopack, TypeScript, `src/` dir, `@/*` import alias)
- **React 19.2.4**
- **Tailwind CSS v4**
- **shadcn/ui** (`base-nova` style, built on **Base UI** — not Radix; see note below) + **lucide-react** icons
- **recharts** — charts (revenue area/line charts, donut breakdowns, radial gauges, sparklines)
- **@tanstack/react-table** — the generic `DataTable`
- **next-themes** — light/dark mode toggle
- **axios** — HTTP client
- **zustand** — auth state (persisted to `localStorage`)
- **js-cookie** — reads/writes the JWT cookie that `proxy.ts` checks for route protection

> ⚠️ **This shadcn setup uses [Base UI](https://base-ui.com), not Radix.** Composition uses the
> `render` prop, not `asChild`:
> ```tsx
> // Base UI (this project) — do this:
> <SheetTrigger render={<Button>Open</Button>} />
> // NOT the Radix pattern:
> <SheetTrigger asChild><Button>Open</Button></SheetTrigger>
> ```
> Every trigger/item component generated by `npx shadcn@latest add ...` in this project
> (`DropdownMenuTrigger`, `DropdownMenuItem`, `SheetTrigger`, `SheetClose`, `SidebarMenuButton`,
> etc.) follows this convention — check `components/ui/*.tsx` before assuming Radix APIs apply.

> ⚠️ **recharts + Turbopack SSR crash**: importing any `recharts` component from a page/component
> that gets server-rendered crashes `next build` with `TypeError: (0, k.createContext) is not a
> function` during "Collecting page data". Every recharts-based component
> (`stat-card.tsx`, `chart-card.tsx`, `donut-legend-card.tsx`, `radial-gauge-card.tsx`,
> `revenue-comparison-chart.tsx`, `data-table/mini-trend.tsx`) is therefore only ever imported via
> `next/dynamic(..., { ssr: false })` at its usage site (see `dashboard/page.tsx`,
> `analytics/page.tsx`, `orders/page.tsx`). If you add a new chart, follow the same pattern —
> don't import recharts components directly into a page.

> ⚠️ **Next.js 16 breaking change**: `middleware.ts` has been renamed to **`proxy.ts`** (the
> exported function is `proxy`, not `middleware`). This project already uses `src/proxy.ts`.
> If you're used to older Next.js docs/patterns, read `node_modules/next/dist/docs/` (see
> `AGENTS.md`) before changing anything routing/proxy related — several APIs differ from what
> you may expect from earlier Next.js versions.

## Getting started

```bash
npm install
npm run dev
```

The dev server runs on **http://localhost:3001** (not 3000) because the backend
(`../backed`) already listens on port 3000 (see `../backed/.env`). Both `dev` and `start`
scripts in `package.json` are pinned to `-p 3001` to avoid the conflict.

Copy `.env.example` to `.env.local` if it doesn't already exist, and point it at your backend:

```
NEXT_PUBLIC_API_BASE_URL=http://localhost:3000/api
```

## How the pieces fit together

1. **`lib/http.ts`** — single axios instance. Base URL from `NEXT_PUBLIC_API_BASE_URL`. A
   request interceptor reads the JWT from the `admin_token` cookie and sets
   `Authorization: Bearer <token>`. A response interceptor clears the session and redirects to
   `/login` on `401`.
2. **`lib/createCrudService.ts`** — most admin resources are plain REST CRUD
   (`GET /`, `GET /:id`, `POST /`, `PUT /:id`, `DELETE /:id`), so this factory avoids repeating
   the same axios calls 40+ times. Resources with different endpoints (e.g. `user.service.ts`,
   `dashboard.service.ts`) are written by hand instead of forced into the factory.
3. **`store/auth.store.ts`** — holds `{ admin, isAuthenticated }`, persisted to `localStorage`
   so a page refresh doesn't log the admin out. The JWT itself lives in a cookie (not
   localStorage) so `proxy.ts` — which runs on the server — can read it.
4. **`proxy.ts`** — redirects to `/login` if there's no `admin_token` cookie on any route other
   than `/login`, and redirects away from `/login` back to `/dashboard` if a token is already
   present. This is a presence check only (fast, no DB hit); the backend still verifies the
   token's validity via `adminAuthMiddleware` on every request.

## Adding a new API-backed page

```ts
// 1. Already have a service? Just import it:
import { categoryService } from "@/services";

const categories = await categoryService.list({ page: 1, limit: 20 });

// 2. New backend resource that's plain CRUD? One line:
export const newResourceService = createCrudService("/admin/new-resource");

// 3. New resource with custom endpoints? Write it by hand next to the others,
//    following the pattern in user.service.ts or dashboard.service.ts.
```

## Known gaps / next steps

- **Every `(admin)` page reads from `lib/mock/*` static fixtures, not the real API.** Add/Edit/Delete
  on list pages mutate local React state via `useState` only (so the UI feels interactive) — nothing
  is persisted or sent to the backend. Wiring a page to the real API means replacing the
  `useState(initialX)` call with the matching `services/*.service.ts` call (already built and
  working — see the "API-backed page" section above) and moving the local mutations into
  `.then()`/`await` calls against that service.
- The JWT cookie is set client-side via `js-cookie` (not `httpOnly`). Fine for now; if this
  needs hardening later, move login through a Next.js Route Handler that sets an `httpOnly`
  cookie instead of returning the token to client JS.
- `admin.auth` only exposes `POST /login` on the backend today — no refresh-token or
  admin "me"/profile endpoint exists yet, so there's nothing to call for those.
- Reference design: an "Apex Dashboard" shadcn/Next.js admin template (dark sidebar, green accent,
  stat cards with sparklines, tabbed area charts) — colors/spacing were approximated by hand in
  `app/globals.css`, not pulled from a purchased template file.
